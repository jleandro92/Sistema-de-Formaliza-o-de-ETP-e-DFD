✅ Requisitos Funcionais (RF)

👉 O que o sistema deve fazer

🔐 Autenticação e Usuários

RF01 – O sistema deve permitir o cadastro de usuários.

RF02 – O sistema deve permitir autenticação por login e senha.

RF03 – O sistema deve permitir níveis de acesso (usuário comum, gestor, licitações).

RF04 – O sistema deve permitir a edição dos dados do usuário.

📋 ETP – Estudo Técnico Preliminar

RF05 – O sistema deve permitir criar um ETP.

RF06 – O sistema deve permitir editar um ETP.

RF07 – O sistema deve permitir listar ETPs cadastrados.

RF08 – O sistema deve permitir salvar ETPs em modo rascunho.

RF09 – O sistema deve permitir preencher todos os campos obrigatórios do ETP.

RF10 – O sistema deve permitir associar um mapa de riscos ao ETP.

⚠️ Mapa de Riscos

RF11 – O sistema deve permitir cadastrar riscos associados ao ETP.

RF12 – O sistema deve permitir definir probabilidade e impacto do risco.

RF13 – O sistema deve permitir registrar ações preventivas e de contingência.

RF14 – O sistema deve permitir identificar os responsáveis pelos riscos.

📑 DFD – Documento de Formalização da Demanda

RF15 – O sistema deve permitir criar um DFD.

RF16 – O sistema deve permitir editar um DFD.

RF17 – O sistema deve permitir listar DFDs cadastrados.

RF18 – O sistema deve permitir relacionar um DFD a um ETP.

RF19 – O sistema deve permitir cálculo automático dos valores totais do DFD.

📝 Documentos e Versionamento

RF20 – O sistema deve permitir gerar documentos ETP e DFD em formato .docx.

RF21 – O sistema deve manter histórico de versões dos documentos.

RF22 – O sistema deve permitir consultar versões anteriores.

RF23 – O sistema deve registrar data e autor de cada alteração.

🔍 Consulta e Relatórios

RF24 – O sistema deve permitir a consulta de documentos por filtros.

RF25 – O sistema deve permitir visualização dos documentos cadastrados.
